Q01
SUMBasic
+
Calculate Total Monthly Sales
You have daily sales figures in cells B2:B32. Your manager wants a total for the month.
Formula
=SUM(B2:B32)
SUM adds all values in a range. Select the range B2:B32 to cover all 31 days. Place this in cell B33 for a clean total row.
Press Alt+= to auto-insert SUM for selected range above.

AVERAGEBasic
+
Find Average Employee Score
HR has performance review scores in C2:C50. Find the department average for the annual report.
Formula
=AVERAGE(C2:C50)
AVERAGE calculates the arithmetic mean. It automatically ignores empty cells but includes zeros, so ensure blank cells aren't entered as 0.
Use AVERAGEIF to average only scores above a threshold, e.g., passing scores.

Pass/Fail Student Results
Exam scores are in column D. Students pass if they score 40 or above. Show "Pass" or "Fail" in column E.

IF(condition, value_if_true, value_if_false). If D2 is 40 or more → "Pass", otherwise → "Fail". Drag down to apply to all students.
Drag the fill handle (bottom-right corner) down to copy the formula to all rows.

Q11
VLOOKUPLookup
+
Fetch Employee Department from HR Table
Employee IDs are in column A of your sheet. The HR master table (Sheet2, A:C) has ID, Name, Department. Pull the department into your report.
Formula
=VLOOKUP(A2,Sheet2!$A:$C,3,FALSE)
VLOOKUP(lookup_value, table, col_index, [exact_match]). A2 = ID to find, Sheet2!$A:$C = the lookup table, 3 = return the 3rd column (Department), FALSE = exact match.
Always use FALSE for exact matches. TRUE does approximate match, suitable only for sorted numeric data.
Q12
COUNTIFCount
+
Count Orders by Region
Sales orders have a "Region" column (column C). Count how many orders came from the "North" region.
Formula
=COUNTIF(C2:C500,"North")
COUNTIF(range, criteria) counts cells matching a condition. The criteria "North" is case-insensitive. You can also reference a cell: =COUNTIF(C2:C500,F1) where F1 contains "North".
Use wildcards: "North*" matches "North", "Northeast", "Northwest" etc.
Q13
SUMIFConditional
+
Total Sales for One Salesperson
Column B has salesperson names, column D has order values. Sum all sales made by "Priya Sharma".
Formula
=SUMIF(B2:B500,"Priya Sharma",D2:D500)
SUMIF(range, criteria, sum_range) — checks column B for the name, then sums the corresponding values in column D. The sum_range must align row-by-row with the range.
Reference a cell instead of hardcoding the name to make the formula dynamic: =SUMIF(B:B,G1,D:D)
Q14
SUMIFSMulti
+
Sales for One Salesperson in One Region
Filter sales by BOTH salesperson (col B) AND region (col C) to get a specific total for quarterly review.
Formula
=SUMIFS(D2:D500,B2:B500,"Priya Sharma",C2:C500,"North")
SUMIFS(sum_range, range1, criteria1, range2, criteria2, ...) supports multiple conditions (up to 127). All conditions must be true for the row to be included in the sum.
SUMIFS is the multi-condition upgrade of SUMIF — use it whenever you need 2+ filters.
Q15
INDEX/MATCHLookup
+
Look Up Price by Product Code (Left Lookup)
Your price table has Product Code in column C, Price in column A. VLOOKUP can't look left — use INDEX/MATCH instead.
Formula
=INDEX(A2:A200,MATCH(G2,C2:C200,0))
MATCH(G2,C2:C200,0) finds the row position of the code. INDEX(A2:A200,...) returns the value at that row from column A. This works for lookups in any direction.
In Excel 365/2021, use XLOOKUP instead — it's simpler and more powerful than both VLOOKUP and INDEX/MATCH.

XLOOKUP365
+
Two-Way Product-Price Matrix Lookup
You have a pricing matrix: products in rows (A2:A20), customer tiers in columns (B1:F1). Find the price for a specific product + tier combo.
Formula
=XLOOKUP(H1,A2:A20,XLOOKUP(H2,B1:F1,B2:F20))

Grade Students Based on Score Range
Scores in column B: 90+ = A, 75-89 = B, 60-74 = C, 45-59 = D, below 45 = F. Build a grade formula.

PMTFinance
+
Calculate Monthly Loan EMI
Home loan: ₹50 lakh principal, 8.5% annual interest, 20-year tenure. Calculate the monthly EMI for the customer.
Formula
=PMT(8.5%/12, 20*12, -5000000)

Create a Custom Reusable Tax Calculator Function
Your team needs a named function TAX_CALC that takes gross income and tax rate, returning net after tax. Make it reusable across the workbook.

LETPerformance

Optimize Complex Bonus Calculation
Bonus = (Sales − Target) × Rate × Multiplier. Sales in B2, Target in C2, Rate in D2, Multiplier based on region in E2. Build a clean, readable formula.

Q34
NPV/IRRFinance
+
Evaluate Project Viability with NPV and IRR
Initial investment: -₹1,00,000 in B1. Cash flows for years 1–5 in B2:B6. Discount rate: 10%. Calculate NPV and IRR for board presentation.
Formulas
NPV: =B1+NPV(10%,B2:B6)  |  IRR: =IRR(B1:B6)
NPV discounts future cash flows at the given rate (add B1 separately as it's Year 0). IRR finds the rate that makes NPV=0. If IRR > cost of capital → invest. If NPV > 0 → value-adding.
XNPV and XIRR use actual dates for uneven payment periods — more accurate for real projects.

SEQUENCEDynamic
+
Generate Automatic Project Timeline Dates
Project starts on a date in A1, runs for 52 weeks. Auto-generate all Monday dates for the project calendar header row.
Formula (spills 52 dates)
=A1+SEQUENCE(1,52,0,7)

Auto-Send Report Email When Data is Refreshed
Each morning after Power Query refresh, auto-email the summary report to your manager via Outlook using VBA.

Sub SendReport() Dim ol As Object, mail As Object Set ol = CreateObject("Outlook.Application") Set mail = ol.CreateItem(0) mail.To = "manager@company.com" mail.Subject = "Daily Sales Report - " & Format(Date,"DD-MMM-YYYY") mail.Body = "Please find attached today's report." mail.Attachments.Add ThisWorkbook.FullName mail.Send End Sub

+
Optimize Product Mix to Maximize Profit
You make 4 products. Each has different profit margins and consumes different amounts of 3 limited resources. Find the optimal production quantities.
Using Excel Solver
Data → Solver → Set Objective: Profit cell (Max) → By Changing: Quantity cells → Add Constraints: Resource usage ≤ capacity, Quantities ≥ 0 → Solve

BYROW365
+
Score Each Salesperson Row Across Multiple KPIs
20 salespersons, 5 KPIs (B2:F21). Score each salesperson as the average of their normalized KPI scores in one formula without helper columns.
Formula (spills 20 results)
=BYROW(B2:F21, LAMBDA(row, AVERAGE(row)/MAX(B2:F21)))
BYROW applies a LAMBDA function to each row of a range, returning one result per row. Combined with LAMBDA, it's extremely powerful for row-wise calculations that previously required helper columns or array formulas.
Companion function BYCOL does the same column-wise — useful for per-KPI analysis across all employees.
